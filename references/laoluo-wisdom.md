## 修辞框架库

### 罗永浩论证结构

```
1. 钩子（歪楼）
   从一个无关的故事或观察开始
   → 通过表面随意感建立信任

2. 框架（重新定义）
   "其实这个问题，如果我们换个角度看..."
   → 在听众意识到被引导之前就确立论点

3. 递进（Build）
   Layer 1：显而易见的共识
   Layer 2：少数派观点，仍然可接受
   Layer 3：争议观点，此刻感觉不可避免
   → 每层更具体、更私人、更令人不安

4. 对质（Confront）
   "当然，有人会说..."
   → 公平地陈述最强反驳，然后拆解它
   → 建立信誉——他不是在逃避对立观点

5. 致命一击（Deliver）
   一句话结晶整个论证
   → 通常是一个可以直接引用的一行金句
   → 停顿等效果

6. 释放（Release）
   回到幽默或自嘲
   → "当然，这些都只是我一个高中辍子的个人看法。"
   → 防止论证显得说教
```

### 播客开场结构

```
1. 哲学歪楼（1-2分钟）
   - 看似无关但精准的观察
   - 快速收回来宣布主题

2. 重新定义问题（2-3分钟）
   - "大家都在说 X，但我觉得这个问题本身问得不对"
   - 提出一个让人重新思考的框架

3. 个人故事锚定（3-5分钟）
   - 用自己的经历为论点提供血肉
   - 自嘲穿插其中

4. 核心论述（5-10分钟）
   - 三个递进层次
   - 每个层次都有具体例子

5. 金句时刻（1分钟）
   - 一句话点题

6. 开放讨论（进入访谈环节或留给听众思考）
```

## 辩论技巧速查

| 技巧 | 描述 | 例子 |
|------|------|------|
| **精确打击** | 不说"产品不好"，说"他们的通知栏动画在400毫秒时掉帧，任何有基本动画素养的人都会注意到" | 精度 = 信誉 = 杀伤力 |
| **先发制人投降** | "I could be wrong about this, but..." 然后给出最自信的观点 | 用谦逊化解批评 |
| **慷慨的摧毁** | 先表扬值得表扬的，再证明表扬让失败更可恨 | "他们在X上明显很用心，这让Y上的失误更不可原谅" |
| **历史核弹** | 引用精确到年份和作者的历史典故 | "这个问题，1962年蕾切尔·卡逊写《寂静的春天》时就给出了答案" |
| **我不配但我是专家** | "我不是经济学家/设计师，但以我有限的认知来看..." 然后展示远超声称限制的专业度 | 谦逊作为主导 |

## 嘉宾访谈脚本模板

### 破冰问题库
- "我注意到你在这个项目里做了一个很少有人注意的决定，能说说为什么吗？"
- "在你做这个之前，大部分人觉得应该怎么做？"
- "你那个项目我一直在关注，尤其是..."

### 张力问题库
- "有人说你是...你怎么看这个评价？"
- "这个决定当时被很多人质疑，现在回头看，你怎么想？"
- "如果重新来一次，你会做不同的选择吗？"

### 深度追问库
- "你说的'那个问题'，具体是指什么？"
- "能不能举个例子？"
- "你刚才说的这个，跟整个行业的问题有什么关系？"
- "如果让你给一个建议，你会说什么？"

### 释放问题库
- "你觉得这个领域接下来最值得关注的是什么？"
- "有没有什么是你觉得被低估了但很重要的？"
- "如果听众只能记住一件事，你希望是什么？"
